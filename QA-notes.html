Profile Review — Feedback Notes

QA observations · SP + Company profiles · March 4, 2026

SP Profile — Skilled Professional
Copy Issue SP-PE-S5 Work Preferences — wrong-side phrasing
The question reads "What kind of work are you looking to hire for?" — that phrasing belongs on the Company side since companies hire, not skilled professionals. Flagging in case it slipped through.
General / Cross-Profile
Behaviour PROFILE Menu auto-closes on page refresh
The Menu on profile pages collapses every time the page refreshes, requiring the user to manually open it again. Since we're approved, it should be set to remain auto-expanded by default.
Behaviour SIGNUP "Save Successful" message fires on every step navigation
A "Save Successful" toast appears at the top of the screen every time the user moves between steps during signup. Not sure if intentional — it may be expected to appear only when the user manually saves something. Worth confirming.
Company Profile
Redundancy CP-PE-S2 / CP-PE-S3 "Are you hiring?" — duplicate question
"Are you hiring?" appears in General Information (CP-PE-S2), and "Are you currently hiring?" appears again in the Aircraft section (CP-PE-S3). These seem redundant — one of them should likely be removed.
Copy Issue CP-PE-S5 "For Hire" label — misleading on Company profile
"For Hire" implies someone is available to be hired, which fits a Skilled Professional but not a Company. On the Company profile this should probably read something like "Currently Hiring" or "Actively Hiring" to better reflect that the company is the one offering work.
UX Gap CP-PE-S3 No option to remove an aircraft entry
Once a user adds an aircraft and saves, there is no way to delete it. If someone accidentally clicks "Add Another Aircraft" and saves, they're stuck with that extra entry. A remove / delete button per aircraft entry seems like a must-have.
